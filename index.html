Conclusiones

Conclusiones del Ejemplo 1 y Ejercicios

  • Validez y Flexibilidad de XML: Trabajar con el Ejemplo 1 y los ejercicios me ha ayudado a entender mejor cómo crear documentos XML que sean válidos y bien formados. A medida que añadimos nuevas etiquetas y atributos en los ejercicios, vimos cómo XML permite adaptar la estructura de los documentos según nuestras necesidades, sin necesidad de una DTD. Esto demuestra la flexibilidad de XML para manejar diferentes tipos de datos de manera creativa.
  • Importancia de la Estructura: A lo largo de los ejercicios, aprendí lo crucial que es seguir las reglas de XML, como cerrar todas las etiquetas correctamente y evitar el traslape. La estructura jerárquica que se mantiene en los documentos ayuda a asegurar que los datos se presenten de forma organizada y clara.

Conclusiones del Ejemplo 2 y Ejercicios

  • Uso de Atributos y Etiquetas: El Ejemplo 2 nos mostró cómo estructurar información geográfica en XML. Conclusiones de los Ejercicios CSS

  • Aplicación de Estilos a XML: La experiencia de aplicar estilos CSS a documentos XML me mostró cómo mejorar la presentación visual de los datos. Aunque XML por sí solo es una estructura de datos, los estilos CSS permiten que los documentos se vean más atractivos y sean más fáciles de leer.
  • Personalización y Responsividad: Trabajar con diferentes archivos CSS y ajustar estilos me enseñó cómo hacer que los documentos XML sean más responsivos y estéticamente agradables. Aprendí a usar técnicas como media queries para que el diseño se ajuste a diferentes dispositivos y tamaños de pantalla.
  • Flexibilidad en el Diseño: Finalmente, ver cómo se pueden personalizar los estilos CSS para XML me mostró la flexibilidad que tenemos para diseñar documentos según nuestras necesidades específicas. Esto incluye mostrar atributos como contenido y aplicar estilos llamativos para mejorar la presentación.
